# Docker Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Containerizar frontend React/Vite + backend Express + PostgreSQL para rodar em dev com hot-reload e em prod com build otimizado.

**Architecture:** Multi-stage Dockerfile (builder Vite → prod Express); `docker-compose.yml` para prod (app + db); `docker-compose.dev.yml` para dev (frontend + app + db com volumes montados). Em prod, Express serve o bundle `dist/` como estáticos além das rotas `/api`.

**Tech Stack:** Docker Engine 24+, Docker Compose v2, Node 20 Alpine, PostgreSQL 16 Alpine

### Task 2: Dockerfile multi-stage

**Files:**
- Create: `Dockerfile`

- [ ] **Step 1: Criar `Dockerfile`**

```dockerfile
# ── Stage 1: build frontend ──────────────────────────────────
FROM node:20-alpine AS builder
WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY index.html ./
COPY vite.config.js tailwind.config.js postcss.config.js ./
COPY src/ ./src/
COPY public/ ./public/

RUN npm run build

# ── Stage 2: production ──────────────────────────────────────
FROM node:20-alpine AS prod
WORKDIR /app

COPY server/package*.json ./server/
RUN cd server && npm ci --omit=dev

COPY server/ ./server/
COPY --from=builder /app/dist ./dist/

RUN mkdir -p public/images/produtos

EXPOSE 3001
CMD ["node", "server/src/server.js"]
```


### Task 5: vite.config.js — proxy via env var

**Files:**
- Modify: `vite.config.js`

O arquivo atual tem o proxy hardcoded em `http://localhost:3001`. Em Docker dev, o Vite precisa apontar para o serviço `app` via `http://app:3001`.

- [ ] **Step 1: Atualizar `vite.config.js`**

Conteúdo atual:
```js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      '/api': 'http://localhost:3001',
    },
  },
})
```

Novo conteúdo:
```js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      '/api': process.env.VITE_API_TARGET || 'http://localhost:3001',
    },
  },
})
```

- [ ] **Step 2: Verificar dev local sem Docker**

Rodar `npm run dev` fora do Docker deve continuar funcionando normalmente (sem `VITE_API_TARGET` definida, usa `localhost:3001`).

- [ ] **Step 3: Commit**

```bash
git add vite.config.js
git commit -m "chore(docker): vite proxy usa VITE_API_TARGET com fallback localhost"
```

### Task 6: server/src/app.js — serve static dist/ em prod

**Files:**
- Modify: `server/src/app.js`

Em produção, o Express precisa servir os arquivos de `dist/` e redirecionar todas as rotas não-API para `index.html` (SPA routing).

- [ ] **Step 1: Adicionar `require('path')` e bloco de static ao `server/src/app.js`**

Adicionar no topo (após os requires existentes):
```js
const path = require('path');
```

Substituir a linha do handler 404 atual:
```js
app.use((req, res) => res.status(404).json({ error: 'Rota não encontrada.' }));
```

Por:
```js
if (process.env.NODE_ENV === 'production') {
  const distPath = path.join(__dirname, '../../dist');
  app.use(express.static(distPath));
  app.get('*', (req, res) => {
    res.sendFile(path.join(distPath, 'index.html'));
  });
} else {
  app.use((req, res) => res.status(404).json({ error: 'Rota não encontrada.' }));
}
```

- [ ] **Step 2: Verificar sintaxe**

```bash
cd server && node -e "require('./src/app.js'); console.log('OK')"
```

Esperado: `OK` sem erros.

- [ ] **Step 3: Commit**
